第 7 课 · 知识 + 技能

SQLite 持久化——对话历史不再丢失

用一个文件型数据库，让会话跨重启存活

现在的问题

内存存储（现在）
sessionMap 存在 Node.js 进程里。 服务器重启 → 所有历史清空。 用户刷新页面之间断线 → 上下文丢失。
SQLite 存储（目标）
消息写进 chat.db 文件。 重启服务器 → 历史还在。 每次请求从数据库读历史，写完再存回去。

为什么选 SQLite？

SQLite 是一个嵌入式数据库——没有独立服务器进程，数据就是磁盘上的一个文件 （chat.db）。对于个人项目和中小规模应用，它是最省心的选择：

  • 零配置，不用装 MySQL / PostgreSQL
  • 一个文件就是整个数据库，备份 = 复制文件
  • 全球使用量最大的数据库引擎（手机、浏览器内置）

我们用 better-sqlite3——它提供同步 API（不需要 await），让代码更简单直接。

数据怎么存？

不需要 sessionMap，改成一张数据库表——每一行是一条消息：

idsession_idrolecontent
1sess-abc system 你是一个 AI 全栈工程师助教…
2sess-abc user 什么是闭包？
3sess-abc assistant 闭包是函数和它创建时的词法环境的组合…
4sess-abc user 给我一个代码例子

读历史时：SELECT * FROM messages WHERE session_id = ? ORDER BY id
写消息时：INSERT INTO messages (session_id, role, content) VALUES (?, ?, ?)

动手：四步完成迁移

1
安装 better-sqlite3
在项目根目录执行：
npm install better-sqlite3
npm install --save-dev @types/better-sqlite3

安装时会自动编译原生模块，可能需要 1–2 分钟。如果报错提示缺少编译工具，请告知我。

2
新建 src/db.ts——初始化数据库
创建这个文件：
src/db.ts
import Database from 'better-sqlite3'

export const db = new Database('chat.db')

db.exec(`
  CREATE TABLE IF NOT EXISTS messages (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT    NOT NULL,
    role       TEXT    NOT NULL,
    content    TEXT    NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
  )
`)

export const insertMsg = db.prepare(
  'INSERT INTO messages (session_id, role, content) VALUES (?, ?, ?)'
)

export const getHistory = db.prepare(
  'SELECT role, content FROM messages WHERE session_id = ? ORDER BY id ASC'
)

export const countSession = db.prepare(
  'SELECT COUNT(*) AS cnt FROM messages WHERE session_id = ?'
)
3
修改 src/index.ts——用数据库替换 sessionMap

改动三处：

① 顶部新增 import：

import { db, insertMsg, getHistory, countSession } from './db'

② 删掉 sessionMap：

const sessionMap = new Map<string, ChatMessage[]>()

③ 替换路由里的历史读写逻辑：

src/index.ts — /api/chat 路由（完整版）
app.post('/api/chat', async (c) => {
  const { message, sessionId } = await c.req.json()

  if (!message || !sessionId) {
    return c.json({ error: '缺少必填参数：message 和 sessionId' }, 400)
  }

    // 新 session → 先插入 system prompt
    const { cnt } = countSession.get(sessionId) as { cnt: number }
    if (cnt === 0) insertMsg.run(sessionId, 'system', SYSTEM_PROMPT)

    // 写入 user 消息
    insertMsg.run(sessionId, 'user', message)

    // 读取完整历史
    const history = getHistory.all(sessionId) as ChatMessage[]

  try {
    const response = await client.chat.completions.create({
      model: 'deepseek-chat',
      messages: history,
    })
    const reply = response.choices[0].message.content ?? ''
        insertMsg.run(sessionId, 'assistant', reply)
    return c.json({ reply, timestamp: new Date().toISOString() })
  } catch (err) {
        // 失败时撤销刚写入的 user 消息
        db.prepare('DELETE FROM messages WHERE session_id = ? AND role = ? ORDER BY id DESC LIMIT 1').run(sessionId, 'user')
    console.error('AI 调用失败', err)
    return c.json({ error: 'AI 服务暂时不可用，请稍后重试' }, 503)
  }
})
4
重启并验证持久化

重启服务器，发一条消息，再重启一次，然后问 AI "我刚才说了什么"—— 如果它能回答，说明持久化成功了。
