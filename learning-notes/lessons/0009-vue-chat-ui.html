Vue 3 聊天界面——让浏览器能跟 AI 说话

后端加一行 CORS，前端写一个组件，全栈打通

你只需要关注两件事

你已经熟悉 Vue 3，所以本课的重点不是组件语法，而是两个全栈连接点：

  • CORS——为什么浏览器会拦截你的 fetch，怎么解决
  • sessionId 管理——前端怎么维护会话身份

CORS：浏览器的同源检查

你的前端跑在 localhost:5173，后端跑在 render.com——不同源。 浏览器在发 fetch 之前，会先发一个 OPTIONS 预检请求，问后端： "你允许来自这个前端的请求吗？"

浏览器
──OPTIONS──▶
后端
◀── 200 + CORS 头 ──
✓ 允许，继续发真实请求
浏览器
──OPTIONS──▶
后端
◀── 无 CORS 头 ──
❌ 请求被浏览器拦截

后端不加 CORS 头，浏览器就会报 CORS policy 错误， 即使后端本身能收到请求、能响应，浏览器也不会把结果交给你的 JS 代码。

注意：CORS 是浏览器的限制，curl 和 test.http 不受它影响—— 所以之前用 test.http 测试都正常，但从浏览器发 fetch 就会被拦截。

动手：四步完成全栈

1
后端加 CORS 中间件（1 行改动）
在 src/index.ts 顶部 import 之后，所有路由之前加：
src/index.ts
import { cors } from 'hono/cors'

// ... 其他 import 和初始化代码 ...

app.use('*', cors())  // 必须在所有路由之前

app.get('/', ...)

cors() 是 Hono 内置的，不需要额外安装包。 改完后 commit push，Render 会自动重新部署。

2
创建 Vue 3 前端项目
在 ai-backend 同级目录下（不是里面）执行：
npm create vite@latest ai-frontend -- --template vue-ts
cd ai-frontend
npm install

前后端放在同级的两个目录里，各自独立管理依赖。

3
配置后端地址（环境变量）
在 ai-frontend/ 根目录新建 .env.local：
VITE_API_URL=http://localhost:3000

本地开发指向本地后端。上线前改成你的 Render 地址即可，代码不用动。

4
写 Chat 组件
把 src/App.vue 替换为以下内容：
ai-frontend/src/App.vue
<script setup lang="ts">
import { ref } from 'vue'

type Message = { role: 'user' | 'assistant'; content: string }

const API_URL = import.meta.env.VITE_API_URL || 'http://localhost:3000'
const sessionId = crypto.randomUUID()  // 每次打开页面生成新会话

const messages = ref<Message[]>([])
const input = ref('')
const loading = ref(false)

async function send() {
  const text = input.value.trim()
  if (!text || loading.value) return

  messages.value.push({ role: 'user', content: text })
  input.value = ''
  loading.value = true

  try {
    const res = await fetch(`${API_URL}/api/chat`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ message: text, sessionId }),
    })
    const data = await res.json()
    if (!res.ok) throw new Error(data.error)
    messages.value.push({ role: 'assistant', content: data.reply })
  } catch {
    messages.value.push({ role: 'assistant', content: '出错了，请稍后重试' })
  } finally {
    loading.value = false
  }
}
</script>

<template>
  <div class="chat">
    <div class="messages">
      <div
        v-for="(msg, i) in messages"
        :key="i"
        :class="`bubble ${msg.role}`"
      >{{ msg.content }}</div>
      <div v-if="loading" class="bubble assistant loading">…</div>
    </div>
    <div class="input-row">
      <input
        v-model="input"
        placeholder="输入消息…"
        @keydown.enter="send"
        :disabled="loading"
      />
      <button @click="send" :disabled="loading">发送</button>
    </div>
  </div>
</template>

启动方式：npm run dev，打开 localhost:5173。

验证方法： 同时启动后端（npm start）和前端（npm run dev）， 在浏览器界面里发一条消息，能收到 AI 回复就全栈打通了。 如果看到 CORS 报错，说明后端的 app.use('*', cors()) 没有加或没有部署。

关于 sessionId

crypto.randomUUID() 在组件顶层执行一次， 每次刷新页面会生成新的 ID——刷新 = 开始新会话。 这是最简单的策略，适合当前阶段。

如果你想让同一个用户跨刷新保持会话，可以把 sessionId 存到 localStorage： 

const sessionId = localStorage.getItem('sessionId') ?? (() => {
  const id = crypto.randomUUID()
  localStorage.setItem('sessionId', id)
  return id
})()
