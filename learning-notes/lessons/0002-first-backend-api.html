10 分钟内让 HTTP 服务器跑起来，接收并响应请求

从你已知的出发

你用 Axios 发过无数次请求。你一直是请求的发起方（前端）。 这节课，你要变成请求的接收方（后端）。

我们用 Hono 作为后端框架——它是 TypeScript-first 的， 路由写法和 Vue Router 非常相似，你会立刻感觉熟悉。

动手：创建第一个后端项目

前提：确认已安装 Node.js 18+。 在终端运行 node -v，看到版本号就可以继续。 如果没有安装，先到 nodejs.org 下载 LTS 版本。
1
创建项目目录
mkdir ai-backend
cd ai-backend
npm init -y

npm init -y 生成 package.json，和前端项目一样。

2
安装依赖
npm install hono
npm install -D typescript @types/node tsx

hono 是框架；tsx 让你直接运行 TypeScript 文件，不用手动编译。

3
写第一个接口

创建 src/index.ts，粘贴以下代码：

src/index.ts
import { serve } from '@hono/node-server'
import { Hono } from 'hono'

const app = new Hono()

// GET / — 健康检查，访问根路径
app.get('/', (c) => {
  return c.json({ message: 'AI 后端运行中 ✓' })
})

// POST /api/chat — 接收用户消息，返回（假的）AI 回复
app.post('/api/chat', async (c) => {
  const { message } = await c.req.json()
  return c.json({
    reply: `你说了：${message}（真实 AI 回复下节课接入）`,
    timestamp: new Date().toISOString(),
  })
})

// 启动服务器，监听 3000 端口
serve({ fetch: app.fetch, port: 3000 }, (info) => {
  console.log(`服务器已启动：http://localhost:${info.port}`)
})
4
安装 Node 适配器并运行
npm install @hono/node-server
npx tsx src/index.ts

看到 服务器已启动：http://localhost:3000 就成功了。

5
测试你的接口

方法 A：浏览器直接访问 http://localhost:3000，看到 JSON 响应。

方法 B：新开一个终端，运行： 

curl -X POST http://localhost:3000/api/chat \
  -H "Content-Type: application/json" \
  -d "{\"message\": \"你好\"}"

方法 C（推荐）：安装 VS Code 插件 REST Client， 创建 test.http 文件，比 Postman 轻量。

看到这个响应，说明你成功了：
{"reply":"你说了：你好（真实 AI 回复下节课接入）","timestamp":"2026-..."}

读懂这段代码

在继续之前，确认你理解了三个核心概念：

app.get / app.post：定义路由——当浏览器或前端发来 GET/POST 请求时，执行括号里的函数。和 Vue Router 的 router.get 逻辑一样，只是处理 HTTP 而不是页面跳转。
c.req.json()：读取请求体（request body），就是前端 axios.post(url, data) 里的 data。await 是因为它是异步的。
c.json({...})：返回 JSON 响应，就是前端 axios 拿到的那个对象。
