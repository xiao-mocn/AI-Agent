第 8 课 · 知识 + 技能

部署到互联网——让别人能访问你的 AI

三处代码改动 + 一次 Git Push，应用上线

本课做什么

现在你的 AI 接口只在 localhost:3000 上跑。别人访问不了，换台电脑也没有。 这节课把它部署到 Railway——一个 PaaS（平台即服务）， 你只管推代码，平台负责运行、重启、给你一个公开 HTTPS 域名。

你的浏览器
→
Railway（公网 HTTPS）
→
你的 Node.js 进程
→
DeepSeek API
Railway vs 自己买 VPS： VPS 你需要自己装 Node.js、配置进程守护（pm2）、申请 HTTPS 证书、处理重启。 Railway 这些全帮你做了——你只需要一个 npm start 脚本。

第一步：三处代码改动

在推代码之前，先让项目"告诉"Railway 怎么运行它。

1
package.json — 加 start 脚本和 engines
Railway 执行 npm start 启动服务，engines 告诉它 Node 版本要求：
package.json
{
  "scripts": {
    "start": "tsx src/index.ts",
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "engines": { "node": ">=20" },
  ...
}
2
src/index.ts — PORT 改为动态读取
Railway 会把分配的端口放进 process.env.PORT，写死 3000 会导致外部访问不了：
src/index.ts — 最后几行
serve({ fetch: app.fetch, port: 3000 }, (info) => {
serve({ fetch: app.fetch, port: Number(process.env.PORT) || 3000 }, (info) => {
  console.log(`服务器已启动：http://localhost:${info.port}`)
})
3
src/db.ts — 数据库路径改为可配置
Railway 的磁盘默认是临时的，重启后文件消失。 通过环境变量 DB_PATH 指向挂载的持久化存储（稍后配置）：
src/db.ts — 第一行
export const db = new Database('chat.db')
export const db = new Database(process.env.DB_PATH || 'chat.db')

改完后 commit 并 push 到 GitHub。

第二步：在 Railway 创建项目

  1. 访问 railway.app，用 GitHub 账号登录
  2. 点击 New Project → Deploy from GitHub repo
  3. 选择你的仓库（AI-Gent），点 Deploy Now
  4. Railway 会自动检测到 Node.js 项目，开始构建
关于费用：Railway 注册后有 $5 赠金（Trial plan），用完需要升级到 Hobby 计划（$5/月）。 如果想完全免费，可以改用 Render.com（免费计划，步骤基本相同，但冷启动较慢）。

第三步：配置环境变量

打开 Railway 项目 → 点击你的服务 → 左侧菜单 Variables， 添加以下两个变量：

变量名值说明
DEEPSEEK_API_KEY 你的 API Key 从 .env 文件复制过来
DB_PATH /data/chat.db 指向持久化 Volume（下一步配置）
绝对不要把 .env 文件推进 git 仓库。API Key 一旦泄露，所有费用由你承担。 Railway 的 Variables 界面是放密钥的正确地方——它们加密存储，不会出现在代码里。

第四步：挂载 Volume（让数据库持久化）

Railway 的文件系统默认在每次重启/部署后重置，chat.db 会丢失。 用 Volume 挂载一块持久化磁盘来解决：

  1. 打开你的服务 → 左侧菜单 Volumes → Create Volume
  2. Mount Path 填写 /data
  3. 点击 Create——Railway 会自动重新部署，/data 目录现在是持久化的

这样 /data/chat.db 就不会因为重启而丢失了。

第五步：获取公开 URL 并验证

打开服务 → Settings → Networking → Generate Domain， Railway 会生成一个类似 ai-gent-production.up.railway.app 的域名。

验证方法：把 test.http 里的 http://localhost:3000 换成你的 Railway 域名，发一条消息，确认能收到 AI 回复。 能收到就说明部署成功了。
