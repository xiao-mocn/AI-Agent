第十三课：前端登录页面

Vue Router · 登录/注册 · JWT 存储与携带

目标

后端已经有了注册和登录接口。这节课让前端也能用： 做一个登录/注册页，成功后把 JWT 存进 localStorage， 之后进入聊天页时自动带上 token。

有 token？

聊天页

第一步：安装 Vue Router frontend

# 在 ai-frontend 目录下
npm install vue-router

第二步：创建两个页面组件

新建 src/views/LoginView.vue：

<template>
  <div class="auth-wrap">
    <div class="auth-box">
      <h2>{{ isLogin ? '登录' : '注册' }}</h2>
      <input v-model="username" placeholder="用户名" />
      <input v-model="password" placeholder="密码" type="password" />
      <p v-if="error" class="error">{{ error }}</p>
      <button @click="submit">{{ isLogin ? '登录' : '注册' }}</button>
      <p class="toggle" @click="isLogin = !isLogin">
        {{ isLogin ? '没有账号？去注册' : '已有账号？去登录' }}
      </p>
    </div>
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue'
import { useRouter } from 'vue-router'

const API_URL = import.meta.env.VITE_API_URL || 'http://localhost:3000'
const router = useRouter()
const isLogin = ref(true)
const username = ref('')
const password = ref('')
const error = ref('')

async function submit() {
  error.value = ''
  const path = isLogin.value ? '/login' : '/register'
  const res = await fetch(`${API_URL}${path}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ username: username.value, password: password.value }),
  })
  const data = await res.json()
  if (!res.ok) { error.value = data.error; return }

  if (isLogin.value) {
    // 登录成功：存 token，跳转聊天页
    localStorage.setItem('token', data.token)
    router.push('/')
  } else {
    // 注册成功：切换到登录
    isLogin.value = true
  }
}
</script>

第三步：配置路由 frontend

新建 src/router/index.ts：

import { createRouter, createWebHistory } from 'vue-router'
import LoginView from '../views/LoginView.vue'
import ChatView from '../views/ChatView.vue'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    { path: '/', component: ChatView },
    { path: '/login', component: LoginView },
  ],
})

// 路由守卫：没有 token 就跳转登录页
router.beforeEach((to) => {
  const token = localStorage.getItem('token')
  if (!token && to.path !== '/login') return '/login'
})

export default router

第四步：把 App.vue 的聊天内容移入 ChatView frontend

新建 src/views/ChatView.vue，把现有 App.vue 的全部内容复制进去。 同时把发消息时的 token 来源改为从 localStorage 读： 

// 原来的 prompt 弹框改为直接读 localStorage
const token = localStorage.getItem('token') ?? ''

// headers 里改用这个 token
Authorization: `Bearer ${token}`

第五步：改造 App.vue，挂载路由

把 App.vue 精简为只负责渲染当前路由页面：

<template>
  <RouterView />
</template>

<script setup lang="ts">
import { RouterView } from 'vue-router'
</script>

在 src/main.ts 里注册路由：

import { createApp } from 'vue'
import App from './App.vue'
import router from './router'

createApp(App).use(router).mount('#app')
别忘了后端的鉴权中间件： 上节课的中间件是用 ALLOWED_TOKEN 静态比对，现在要换成 JWT 校验（第十二课第四步的代码）， 确保 /register 和 /login 不被拦截，只拦截 /api/*。

小测验

点击选项查看解析

1. 路由守卫 beforeEach 里为什么要判断 to.path !== '/login'？

  • 提升性能，减少不必要的重定向
  • /login 页面不需要加载 Vue Router
  • 不判断会导致访问 /login 时无限重定向自己
  • 这只是代码风格，不影响功能

没有 token 就跳转 /login，但跳转 /login 时又没有 token……如果不排除 /login， 守卫会无限触发重定向，浏览器报 too many redirects 错误。

2. JWT 存在 localStorage 的主要风险是什么？

  • XSS 攻击可以通过恶意脚本读取 localStorage 里的 token
  • localStorage 有大小限制，存不下 JWT
  • 刷新页面后 localStorage 会被清空
  • Safari 不支持 localStorage

XSS（跨站脚本）：如果页面被注入恶意脚本，攻击者可以 localStorage.getItem('token') 直接拿走 JWT。 更安全的方案是用 HttpOnly Cookie（JS 无法读取），但配置更复杂——当前阶段 localStorage 够用。

3. createWebHistory() 和 createWebHashHistory() 的区别是什么？

  • createWebHistory 只支持现代浏览器
  • createWebHistory 用真实路径（/login），部署时需要服务器把所有路径指向 index.html；Hash 模式用 /#/login，不需要服务器配置
  • createWebHashHistory 性能更好
  • 两者没有区别，只是命名不同

Render Static Site 默认已处理了 SPA fallback，createWebHistory 可以直接用。 如果部署到普通静态服务器没有配置 fallback，刷新 /login 会 404，那时才需要换 Hash 模式。

本节要点

  • Vue Router 的 beforeEach 守卫在每次跳转前运行，可以拦截未登录用户
  • 守卫里必须排除登录页本身，防止无限重定向
  • JWT 存 localStorage，简单但有 XSS 风险；生产级方案用 HttpOnly Cookie
  • App.vue 变成纯壳（只有 <RouterView />），页面逻辑全部放 views/ 下
