第十五课：部署上线

目标时长 1.5 小时 · 难度 ⭐⭐⭐ · 运维/部署

本节课你会解决：
  • Vue Router history 模式在 Render 刷新页面报 404 的问题
  • 前端生产环境变量（VITE_API_URL）确认配置正确
  • 后端生产环境变量（JWT_SECRET、DEEPSEEK_API_KEY）整理到位
  • 推送最新代码后触发自动部署，验证全链路上线

一、为什么刷新页面会 404？

Vue Router 使用 createWebHistory 时，URL 看起来像 https://你的域名/login。

问题在于：浏览器刷新时，这个请求会真的发给服务器，服务器上根本没有 /login 这个文件，所以返回 404。

解决方案：告诉服务器——所有路径都返回 index.html，让前端 JS 自己处理路由。

二、Render 前端 Static Site：配置重写规则

1在前端项目根目录新建 public/_redirects 文件

Render Static Site 支持 Netlify 格式的重定向规则：

/*    /index.html   200

这一行的意思：所有路径（/*）都返回 index.html，状态码 200（不是 301 重定向）。

文件放哪里？ 放在 ai-frontend/public/_redirects。Vite 构建时会把 public/ 下的文件原样复制到 dist/，Render 部署后会自动读取 dist/_redirects。
2验证文件结构 
ai-frontend/
├── public/
│   └── _redirects        ← 新建这个文件
├── src/
└── vite.config.ts

三、后端环境变量清单

登录 Render 后端服务的 Environment 页面，确认以下变量都已配置：

变量名示例值说明
DEEPSEEK_API_KEYsk-xxxDeepSeek API 密钥
JWT_SECRET随机长字符串签名 JWT，必须保密，生产环境不能用 dev-secret
PORT3000（Render 自动注入）通常不需要手动设置
JWT_SECRET 生产规范：不能用 dev-secret，要用随机生成的长字符串。可以在终端运行： 
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
把输出的 64 位十六进制字符串粘贴到 Render 的 JWT_SECRET 环境变量里。

四、前端环境变量确认

1Render 前端 Static Site 的 Environment 页面
变量名示例值
VITE_API_URLhttps://你的后端域名.onrender.com
关键提醒：VITE_* 变量是在构建时打包进 JS 的，不是运行时读取。改完环境变量后必须重新触发构建（点 Manual Deploy 或推送新代码），否则不生效。

五、推送代码，触发自动部署

1推送到 main 分支 
git push origin main

Render 检测到 GitHub 推送后会自动触发构建和部署（约 2~5 分钟）。

2观察部署日志

在 Render 控制台 → Events 标签页查看构建日志，确认：

  • 后端：服务器已启动：http://localhost:xxxx
  • 前端：dist/index.html 出现在构建输出里

六、全链路验证清单

  1. 访问前端域名根路径 / → 自动跳转 /login ✓
  2. 在 /login 直接刷新浏览器 → 不报 404，仍显示登录页 ✓
  3. 注册新用户 → 提示注册成功 ✓
  4. 登录 → 跳转到聊天页 ✓
  5. 在聊天页刷新 → 不报 404，仍显示聊天页 ✓
  6. 发送消息 → AI 正常流式回复 ✓
  7. 点击退出登录 → 跳回登录页，token 清除 ✓
  8. 手动清除 localStorage 中的 token，刷新聊天页 → 自动跳转登录页 ✓

思考题

Q1：现在 SQLite 数据库文件 chat.db 存在 Render 服务器的磁盘上，Render 每次重新部署时磁盘会重置，数据会丢失。生产环境该怎么解决这个问题？

Q2：前端 VITE_API_URL 硬编码了后端地址，本地开发和生产环境切来切去很麻烦。有什么更优雅的方式管理多环境配置？

Q3：Render 免费套餐服务 15 分钟无请求会自动休眠，第一次请求要等 30 秒以上才响应。如何解决这个冷启动问题？

七、本课总结

  • public/_redirects 一行配置解决 history 模式 404 问题
  • 生产环境 JWT_SECRET 必须是随机强密码，VITE_* 变量改完要重新构建才生效
  • 部署后按验证清单逐项检查，确保全链路通畅
