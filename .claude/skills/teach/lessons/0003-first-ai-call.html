第 3 课 · 技能

第一次真正的 AI 对话

把假回复换成真正的 DeepSeek——10 行新代码，见证 AI 回应你

你上节课留下的伏笔

上节课你写了一行占位符： 

reply: `你说了：${message}（真实 AI 回复下节课接入）`

今天就把这行替换掉。你的服务器结构已经完整—— 只需要在 POST /api/chat 里调用 DeepSeek API， 把 AI 的回复塞进 reply 字段就行了。

为什么用 openai SDK 调用 DeepSeek？
DeepSeek 完全兼容 OpenAI 的接口格式，只需要换一个 baseURL。 这意味着你学会这套写法，以后换任何兼容 OpenAI 格式的模型（包括本地模型 Ollama、其他云厂商）都只改一行配置——这是行业通行模式。

新知识：API Key 和环境变量

在第一课你学过：API Key 必须放后端，不能暴露给浏览器。 但后端代码也不能直接写死 Key——代码会提交到 git， 万一仓库泄露，Key 就彻底暴露了。

标准做法是用环境变量：把 Key 写在一个只在本地存在的 .env 文件里， 通过 .gitignore 告诉 git"永远不要提交这个文件"。 代码只读取变量名，不包含 Key 的值。

类比：就像你的银行卡密码—— 你会把密码存在脑子里，而不是写在银行卡背面。 .env 是"脑子"，代码是"银行卡"，.gitignore 确保密码不会跟着卡一起分享出去。

动手：接入 DeepSeek API

前提：你需要一个 DeepSeek API Key。 前往 platform.deepseek.com， 注册后在 API Keys 页面创建一个 Key（格式是 sk-...）。 新账号有免费额度，足够练习使用。
1
安装 openai SDK 和 dotenv
npm install openai dotenv

openai SDK 支持所有兼容 OpenAI 格式的模型，包括 DeepSeek； dotenv 负责把 .env 文件里的变量加载到 Node.js 里。

2
创建 .env 文件（存放 API Key）
.env
DEEPSEEK_API_KEY=sk-你的密钥粘贴在这里

把 sk-你的密钥粘贴在这里 替换成你从控制台复制的真实 Key。 注意等号两边没有空格。

3
创建 .gitignore（防止密钥泄露）
.gitignore
.env
node_modules

这两行确保 git 永远不会提交 .env 和 node_modules。

4
修改 src/index.ts

绿色高亮行是新增的，其余保持不变：

src/index.ts
import 'dotenv/config'                    // 加载 .env 文件
import { serve } from '@hono/node-server'
import { Hono } from 'hono'
import OpenAI from 'openai'                // OpenAI 兼容格式

const app = new Hono()
const client = new OpenAI({
  apiKey: process.env.DEEPSEEK_API_KEY,   // 读取环境变量
  baseURL: 'https://api.deepseek.com',   // 指向 DeepSeek 服务器
})

app.get('/', (c) => {
  return c.json({ message: 'AI 后端运行中 ✓' })
})

app.post('/api/chat', async (c) => {
  const { message } = await c.req.json()

  const response = await client.chat.completions.create({
    model: 'deepseek-chat',              // DeepSeek-V3
    messages: [{ role: 'user', content: message }],
  })

  const reply = response.choices[0].message.content ?? ''

  return c.json({
    reply,
    timestamp: new Date().toISOString(),
  })
})

serve({ fetch: app.fetch, port: 3000 }, (info) => {
  console.log(`服务器已启动：http://localhost:${info.port}`)
})
5
重新启动并测试
npx tsx src/index.ts

然后在 test.http 里发送请求——这次你会收到 DeepSeek 真正生成的回复。

成功标志：回复里的 reply 字段不再是"你说了：…"的固定格式， 而是 DeepSeek 根据你的消息生成的真实自然语言回复。

读懂新增的代码

new OpenAI({ apiKey, baseURL })： baseURL 是关键——它让 SDK 把请求发到 DeepSeek 的服务器，而不是 OpenAI 的服务器。 以后换其他模型（比如本地 Ollama），只需要改这一行。
messages: [{ role: 'user', content: message }]： 这是 LLM API 的通行格式。role 只有两种值：'user'（用户说的）和 'assistant'（AI 说的）。 这个数组就是对话历史——下节课你会在这里放入多轮对话记录。
response.choices[0].message.content： API 返回的是一个 choices 数组（支持同时生成多个候选回复，默认只生成 1 个）。 .content 就是 AI 生成的文字。?? '' 是防止内容为 null 时报错的保险写法。
