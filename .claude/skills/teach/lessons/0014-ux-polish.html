第十四课：前端体验完善

目标时长 1.5 小时 · 难度 ⭐⭐☆ · 纯前端课

本节课你会实现：
  • 接口返回 401 时自动跳转登录页（而不是显示「出错了」）
  • Header 右上角加「退出登录」按钮，清除 token 并跳回登录页
  • 多会话支持：侧边栏可以新建/切换对话，每个对话有独立的 sessionId

一、为什么现在要做这些？

上节课打通了登录 → 拿 token → 带 token 请求的完整链路。但还有两个明显的体验问题：

  1. token 过期没有处理：JWT 7 天后过期，接口会返回 401，前端现在只显示「出错了」，用户不知道要重新登录。
  2. 没有退出登录：用户无法主动退出，token 一直留在 localStorage 里。

第三个功能（多会话）属于加餐，视时间决定是否实现。

二、第一步：401 自动跳登录页

1思路

当前 send() 函数里：

if (!res.ok) throw new Error('请求失败')

所有非 200 状态都抛同一个错误，catch 里显示「出错了」。需要区分 401 和其他错误。

2修改 ChatView.vue

在 send() 函数里，把错误处理改成：

import { useRouter } from 'vue-router'

// script setup 顶部加：
const router = useRouter()

// send() 里把这一行：
if (!res.ok) throw new Error('请求失败')

// 改成：
if (res.status === 401) {
  localStorage.removeItem('token')
  router.push('/login')
  return
}
if (!res.ok) throw new Error('请求失败')

这样 token 失效时会自动清除并跳转登录页，不再显示错误气泡。

四、第三步（加餐）：多会话侧边栏

这部分代码量较多，如果时间有限可以跳过，先把前两步做完。

思路

现在 sessionId 是组件挂载时 crypto.randomUUID() 生成一次，刷新页面就丢失了。多会话需要：

  1. 用一个数组维护多个会话 sessions: { id, title, messages }[]
  2. 左侧显示会话列表，点击切换
  3. 「新建对话」按钮创建新 session
  4. 会话标题用第一条消息的前 15 个字自动生成

数据结构

type Session = {
  id: string
  title: string
  messages: Message[]
}

const sessions = ref<Session[]>([newSession()])
const activeId = ref(sessions.value[0].id)
const activeSession = computed(() =>
  sessions.value.find(s => s.id === activeId.value)!
)

function newSession(): Session {
  return { id: crypto.randomUUID(), title: '新对话', messages: [] }
}

模板结构（加侧边栏）

<div class="app">
  <!-- 侧边栏 -->
  <aside class="sidebar">
    <button class="new-chat-btn" @click="createSession">＋ 新建对话</button>
    <div
      v-for="s in sessions"
      :key="s.id"
      :class="['session-item', { active: s.id === activeId }]"
      @click="activeId = s.id"
    >{{ s.title }}</div>
  </aside>

  <!-- 右侧聊天区域（把原来的 .layout 放这里）-->
  <div class="chat-area">
    <!-- header / messages / footer 保持不变，但 messages 改为 activeSession.messages -->
  </div>
</div>
注意：send() 里的 messages.value 全部改为 activeSession.value.messages，sessionId 改为 activeId.value。

五、验证清单

  1. 手动清除 localStorage 中的 token，刷新聊天页 → 自动跳转登录页 ✓
  2. 登录后，点击「退出登录」→ 跳回登录页，localStorage 的 token 被清除 ✓
  3. （加餐）新建多个对话，每个对话有独立的消息历史，切换不混淆 ✓

思考题

Q1：多会话数据现在只存在内存里，刷新页面就丢失了。如果要持久化到本地，你会用什么方案？

Q2：token 即将过期（比如还剩 1 小时）时，如何在不打断用户操作的前提下自动续签？

Q3：如果多个标签页同时打开，一个标签页退出登录后，其他标签页应该怎么处理？

六、本课总结

  • res.status === 401 单独处理，清 token 并跳登录，而不是走通用错误流程
  • 退出登录 = localStorage.removeItem('token') + router.push('/login')
  • 多会话的核心是用 activeSession computed 把「当前显示的数据」和「路由/点击状态」解耦
