本节课你会实现：
  • 理解为什么生产环境不能用 SQLite
  • 在 Render 上创建免费的 PostgreSQL 数据库
  • 用 pg 包替换 better-sqlite3，支持 SQLite（本地）和 PostgreSQL（生产）双模式

  • 配置环境变量，全链路验证通过

一、为什么不用 SQLite？

SQLite 是一个嵌入式的单文件数据库，适合开发和单用户场景。生产环境有几个致命问题：

特性SQLitePostgreSQL
数据持久性存在本地磁盘，Render 部署重置即丢独立服务，部署不影响数据
并发写入写锁是表级，并发差行级锁，高并发无压力
网络访问仅本地进程可读支持远程连接
扩展能力无主从复制、连接池、备份

简单说：SQLite 是开发工具，PostgreSQL 是生产数据库。

二、整体方案

我们不直接用 PostgreSQL 替换掉 SQLite，而是让代码同时支持两者：

  • 本地开发：继续用 SQLite，零配置，即开即用
  • 生产环境：自动切换到 PostgreSQL

切换依据：DATABASE_URL 环境变量是否存在。有 → PostgreSQL，没有 → SQLite。

三、第一步：在 Render 创建 PostgreSQL 数据库

1创建 PostgreSQL 服务

Render Dashboard → New + → PostgreSQL：

  • Name：ai-backend-db
  • Database：ai_chat（自动生成也行）
  • User：自动生成
  • Region：和你的后端服务选同一个区域（通常是 Oregon）
  • PostgreSQL Version：16
  • Plan：Free（1 GB 存储，10k 行限制，够用）

点击 Create Database，等待几秒创建完成。

2获取连接字符串

创建完成后，在 PostgreSQL Dashboard 找到 Internal Connection String，格式类似：

postgresql://ai_chat_user:xxxxx@dpg-xxxxx.oregon-postgres.render.com/ai_chat
Internal vs External：同一个 Region 的服务之间用 Internal 地址（走 Render 内部网络，更快更安全）。外部工具连接才用 External。
3配置到后端环境变量

在你的后端 Web Service → Environment → 添加：

变量名值
DATABASE_URL上一步的 Internal Connection String

保存后 Render 会自动重新部署。

四、第二步：安装 pg 并改造 db.ts

1安装依赖 
cd ai-backend
npm install pg
npm install -D @types/pg
2改造 db.ts 支持双模式

核心思路：用一个 isPG 标志判断当前用哪套 API，把 SQLite 同步写法改成 async/await。

import Database from 'better-sqlite3'
import { Pool } from 'pg'

// 判断是否使用 PostgreSQL
const DATABASE_URL = process.env.DATABASE_URL
export const isPG = !!DATABASE_URL

// — PG 连接池 —
let pool: Pool
if (isPG) {
  pool = new Pool({ connectionString: DATABASE_URL })
}

// — SQLite 连接 —
const db = new Database(process.env.DB_PATH || 'chat.db')
db.pragma('journal_mode = WAL')

// — 建表（两个数据库都走）—
if (isPG) {
  await pool.query(`
    CREATE TABLE IF NOT EXISTS users (
      id       SERIAL PRIMARY KEY,
      username TEXT UNIQUE NOT NULL,
      password TEXT NOT NULL
    )
  `)
  await pool.query(`
    CREATE TABLE IF NOT EXISTS messages (
      id         SERIAL PRIMARY KEY,
      session_id TEXT NOT NULL,
      role       TEXT NOT NULL,
      content    TEXT NOT NULL,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )
  `)
} else {
  db.exec(`
    CREATE TABLE IF NOT EXISTS messages (
      id         INTEGER PRIMARY KEY AUTOINCREMENT,
      session_id TEXT    NOT NULL,
      role       TEXT    NOT NULL,
      content    TEXT    NOT NULL,
      created_at DATETIME DEFAULT CURRENT_TIMESTAMP
    )
  `)
  db.exec(`
    CREATE TABLE IF NOT EXISTS users (
      id       INTEGER PRIMARY KEY AUTOINCREMENT,
      username TEXT    UNIQUE NOT NULL,
      password TEXT    NOT NULL
    )
  `)
}
PostgreSQL 用 SERIAL 代替 SQLite 的 AUTOINCREMENT，参数占位符用 $1, $2 代替 ?。
3改造查询函数

把原来同步的 db.prepare(...).run() / .get() / .all() 统一改成异步函数：

// ===== 用户相关 =====

export async function createUser(username: string, password: string) {
  if (isPG) {
    await pool.query('INSERT INTO users (username, password) VALUES ($1, $2)', [username, password])
  } else {
    db.prepare('INSERT INTO users (username, password) VALUES (?, ?)').run(username, password)
  }
}

export async function findUser(username: string): Promise<any> {
  if (isPG) {
    const { rows } = await pool.query('SELECT * FROM users WHERE username = $1', [username])
    return rows[0] || null
  }
  return db.prepare('SELECT * FROM users WHERE username = ?').get(username) || null
}

// ===== 消息相关 =====

export async function insertMsg(sessionId: string, role: string, content: string) {
  if (isPG) {
    await pool.query('INSERT INTO messages (session_id, role, content) VALUES ($1, $2, $3)', [sessionId, role, content])
  } else {
    db.prepare('INSERT INTO messages (session_id, role, content) VALUES (?, ?, ?)').run(sessionId, role, content)
  }
}

export async function getHistory(sessionId: string) {
  if (isPG) {
    const { rows } = await pool.query('SELECT role, content FROM messages WHERE session_id = $1 ORDER BY id ASC', [sessionId])
    return rows
  }
  return db.prepare('SELECT role, content FROM messages WHERE session_id = ? ORDER BY id ASC').all(sessionId)
}

export async function countSession(sessionId: string): Promise<number> {
  if (isPG) {
    const { rows } = await pool.query('SELECT COUNT(*) AS cnt FROM messages WHERE session_id = $1', [sessionId])
    return Number(rows[0].cnt)
  }
  const { cnt } = db.prepare('SELECT COUNT(*) AS cnt FROM messages WHERE session_id = ?').get(sessionId) as any
  return cnt
}

// PG 专用：删除 user 消息（出错回滚用）
export async function deleteLastUserMsg(sessionId: string) {
  if (isPG) {
    await pool.query('DELETE FROM messages WHERE session_id = $1 AND role = $2 AND id = (SELECT MAX(id) FROM messages WHERE session_id = $1 AND role = $2)', [sessionId, 'user'])
  } else {
    db.prepare('DELETE FROM messages WHERE session_id = ? AND role = ? ORDER BY id DESC LIMIT 1').run(sessionId, 'user')
  }
}

export async function initDB() {

六、第四步：迁移数据

如果你已经在 SQLite 里有用户和数据，需要迁移到 PostgreSQL。可以写一个迁移脚本：

1创建 src/migrate.ts 
import Database from 'better-sqlite3'
import { Pool } from 'pg'

// 连接两边
const sqlite = new Database('chat.db')
const pg = new Pool({ connectionString: process.env.DATABASE_URL })

async function migrate() {
  console.log('开始迁移...')

  // 建表
  await pg.query(`
    CREATE TABLE IF NOT EXISTS users (
      id       SERIAL PRIMARY KEY,
      username TEXT UNIQUE NOT NULL,
      password TEXT NOT NULL
    )
  `)
  await pg.query(`
    CREATE TABLE IF NOT EXISTS messages (
      id         SERIAL PRIMARY KEY,
      session_id TEXT NOT NULL,
      role       TEXT NOT NULL,
      content    TEXT NOT NULL,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )
  `)

  // 迁移 users
  const users = sqlite.prepare('SELECT * FROM users').all()
  for (const u of users) {
    await pg.query('INSERT INTO users (username, password) VALUES ($1, $2) ON CONFLICT DO NOTHING', [u.username, u.password])
  }
  console.log(`用户迁移完成：${users.length} 条`)

  // 迁移 messages
  const messages = sqlite.prepare('SELECT * FROM messages').all()
  for (const m of messages) {
    await pg.query('INSERT INTO messages (session_id, role, content) VALUES ($1, $2, $3)', [m.session_id, m.role, m.content])
  }
  console.log(`消息迁移完成：${messages.length} 条`)

  await pg.end()
  console.log('迁移完成 ✅')
}

migrate().catch(console.error)
迁移脚本在本地运行，需要本地能连上 Render PostgreSQL。修改后端环境变量后重新部署会自动触发迁移。

八、SQLite vs PostgreSQL SQL 差异总结

功能SQLitePostgreSQL
自增主键INTEGER PRIMARY KEY AUTOINCREMENTSERIAL PRIMARY KEY
参数占位符?$1, $2, $3
当前时间CURRENT_TIMESTAMP（DATETIME 类型）CURRENT_TIMESTAMP（TIMESTAMP 类型）
时间类型DATETIMETIMESTAMP 或 TIMESTAMPTZ
API 风格同步（.run()/.get()/.all()）异步（await pool.query()）

九、本课总结

  • SQLite 适合开发测试，生产环境必须用 PostgreSQL 这类独立数据库
  • Render 自带 PostgreSQL，创建即用
  • 通过 DATABASE_URL 环境变量，实现本地 SQLite / 生产 PostgreSQL 双模式
  • SQLite 和 PostgreSQL 在 SQL 语法和 API 风格上有区别，迁移时需逐项处理
  • 迁移脚本可以帮你把已有数据从 SQLite 导入 PostgreSQL
