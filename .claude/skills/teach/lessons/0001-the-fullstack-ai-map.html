你的 AI 全栈地图

理解你已经站在哪里，以及前方的路

你比你想象的更有优势

"全栈"意味着你能独立掌控一个应用的所有层。 你已经掌握了最难学的部分——用户界面层。 前端开发者转型全栈，比后端开发者转型更顺，因为你已经懂得如何从用户视角思考产品。

"AI 全栈"在这个基础上再加一层：你不只是把 UI 和 API 拼接起来， 还把大型语言模型（LLM）作为应用的"大脑"接进来—— 让它处理自然语言、生成内容、做决策。

一个独立 AI 应用的四层结构

每个可上线、有 AI 能力的完整应用，都可以用这张地图来描述：

①
前端 UI 你已掌握
Vue / React 界面。收集用户输入，向后端发请求，展示 AI 返回的结果。 你的 Axios 调用就在这里出发。
②
后端 API 需要学习
Node.js 服务。处理业务逻辑，保管 API Key，调用 LLM 和数据库。 它是前端和 AI 之间的安全中转站。
③
AI 服务（LLM） 需要学习
Claude / OpenAI 的 API。你的后端把用户的问题组装成 prompt， 发给它，它返回生成的文字。你按 token 数量付费。
④
数据存储 需要学习
PostgreSQL 存用户账号和对话历史（关系数据库）； Vector DB 存语义搜索数据（进阶）。 后端负责读写，前端永远不直接碰数据库。
为什么一定需要后端？
如果你把 OPENAI_API_KEY 写在前端代码里， 任何人打开浏览器开发者工具都能看到。 你的 API 额度会在几小时内被盗刷光。
后端是 API Key 的保险箱。 这是 AI 应用开发第一条不能破的规则。

你的学习路径

对你而言最高效的顺序——充分利用你已有的 TypeScript 经验：

  • Node.js + TypeScript 后端基础
    写 HTTP 接口，你已熟悉 TS，这一步最快
    第 1–2 周
  • 调用 LLM API（Claude / OpenAI）
    让后端和 AI "说话"，立刻有成就感
    第 2–3 周
  • 数据库：PostgreSQL + Prisma
    Prisma 有完整类型提示，前端感觉
    第 3–4 周
  • 部署上线
    Railway / Vercel，给别人一个 URL
    第 4–5 周
  • 进阶：RAG、Tool Use、Agent
    让 AI 读你的文档，自主执行操作
    第 6 周起
每天 1–2 小时，第 2 周末你就能看到一个真实运行的 AI 接口响应你的问题。 第 5 周，你有一个可以给别人看的完整应用。
