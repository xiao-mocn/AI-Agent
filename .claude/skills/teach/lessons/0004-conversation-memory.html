让 AI 记住对话

用 sessionId + 内存 Map 实现多轮对话上下文

现在的问题

你现在的接口每次都像第一次见面——无论你说什么，AI 都不记得之前说过什么：

你
我叫张三
AI
你好，张三！有什么我能帮你的？
你
我叫什么名字？
AI
抱歉，我不知道你的名字，你能告诉我吗？

原因很简单：LLM 本身是无状态的。它不存储任何东西， 每次收到请求，它只看你这次传来的 messages 数组。 要让它"记住"，你必须把历史对话装进这个数组一起传过去。

解决思路

改动只有两处：

  1. 前端：每次请求带上一个 sessionId（标识"这是哪个会话"）
  2. 后端：用一个 Map 按 sessionId 存储历史，每次把历史带上发给 AI，收到回复后追加进去
这节课用内存存对话历史（服务器重启就清空）。 这对本地开发完全够用——等你学到数据库那节课，再把 Map 换成数据库表。 两者的逻辑完全一样，只是换一个存储方式。

动手：改两个文件

1
修改 src/index.ts

绿色高亮行是新增 / 修改的：

src/index.ts
import 'dotenv/config'
import { serve } from '@hono/node-server'
import { Hono } from 'hono'
import OpenAI from 'openai'

const app = new Hono()
const client = new OpenAI({
  apiKey: process.env.DEEPSEEK_API_KEY,
  baseURL: 'https://api.deepseek.com',
})

// 每个 sessionId 对应一段对话历史
type Message = { role: 'user' | 'assistant'; content: string }
const sessions = new Map<string, Message[]>()

app.get('/', (c) => {
  return c.json({ message: 'AI 后端运行中 ✓' })
})

app.post('/api/chat', async (c) => {
  const { message, sessionId } = await c.req.json()

  // 获取或初始化这个会话的历史
  if (!sessions.has(sessionId)) sessions.set(sessionId, [])
  const history = sessions.get(sessionId)!

  // 把用户这句话追加到历史
  history.push({ role: 'user', content: message })

  const response = await client.chat.completions.create({
    model: 'deepseek-chat',
    messages: history,                // 传完整历史，不只是这一句
  })

  const reply = response.choices[0].message.content ?? ''

  // 把 AI 回复也追加到历史
  history.push({ role: 'assistant', content: reply })

  return c.json({ reply, timestamp: new Date().toISOString() })
})

serve({ fetch: app.fetch, port: 3000 }, (info) => {
  console.log(`服务器已启动：http://localhost:${info.port}`)
})

读懂这段逻辑

这就是 AI 聊天应用最核心的数据流，值得在脑子里过一遍：

  1. 用户发来 { message, sessionId }
  2. 后端从 Map 里取出这个 session 的历史数组
  3. 把用户这句话 push 进去（role: 'user'）
  4. 把整个历史数组传给 DeepSeek
  5. 收到 AI 回复，再 push 进去（role: 'assistant'）
  6. 返回给前端
LLM 是无状态的——它每次看到的都是你传来的完整 messages 数组， 靠这个数组"知道"之前说了什么。 你的后端是唯一知道历史的地方。

快速测验 · 三题

1. LLM（如 DeepSeek）本身会存储对话历史吗？

2. 用户 A 和用户 B 同时在线聊天，为什么他们的历史不会混在一起？

3. 重启服务器后，之前的对话历史会怎样？
